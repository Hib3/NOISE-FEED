# Codex 共通指示

- 日本語で簡潔に回答する。
- 不明点は推測せず「不明」と書く。
- Linux / NAS に接続する場合は `ssh nas` を使う。
- NASは低リソース環境として扱う。
- 重い計算、解析、検索、設計判断はWindows側で行う。
- Linux側はファイル参照、作成、編集、プログラム実行、動作確認に使う。
- Linux側で参照した内容は必要範囲だけ要約し、以後の判断はWindows側で行う。
- Linux側での読み取り、書き込み、実行は許可する。
- Linux側では長時間の全体検索、大規模ビルド、重いテスト、大量インストールを避ける。
- まず対象ファイルだけを見る。
- 大きいファイルは丸ごと表示せず、必要範囲だけ読む。
- 必要になるまで全体探索しない。
- 変更は最小差分にする。
